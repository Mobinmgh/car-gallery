# Changelog

One line per builder step: what was built, and any deviation from AGENTS.md and why.

- **Setup** — Initialized git repo and npm project; vendored GSAP 3.15.0 core + ScrollTrigger as plain `<script>` tags (no bundler); self-hosted Vazirmatn (variable, via the `vazirmatn` npm package) and Big Shoulders Display (variable, via `@fontsource-variable/big-shoulders-display`) as woff2, both OFL-licensed; built RTL-aware reset using logical CSS properties; set up the full token system as CSS custom properties in `css/tokens.css`.
- **Section 1 — Nav** — Transparent-over-hero nav, solid charcoal panel + hairline border after 80px scroll; RTL layout (wordmark right, 3 links left, CTA at flow-end/leftmost); mobile collapses to a 3-hairline toggle (not a default icon font) opening a full-width panel, closes on link click or Escape. Deviation: dropped `crossorigin` from the two font `<link rel="preload">` tags — it forced an anonymous CORS-mode fetch that Chromium blocks under `file://` (origin `null`), throwing console errors even though the fonts loaded fine via `@font-face`; same-origin preload doesn't need it here.
- **Section 2 — Hero** — Full-bleed hero with GSAP load-in stagger (eyebrow → headline → sub → CTA → scroll cue, `power3.out`, skipped entirely under `prefers-reduced-motion`); asymmetric right-anchored text column instead of the centered-hero default; CTA breaks from the reading column to sit at the flow-end (left) edge; scroll cue is a literal preview of the route-line signature (dotted hairline + traveling gold pulse) instead of a generic chevron/mouse icon. Background is a CSS graphite/sapphire gradient wash for now — real photo need logged in `IMAGE_PROMPTS.md`. Deviations found while building: (1) `--fs-h1` was originally locked at `clamp(2.5rem, 7vw, 6rem)` in the setup step's token brainstorm, but at 96px the locked headline forced 4-line wraps and crowded the composition — dropped the max to `4.5rem` for a balanced 3-line wrap; (2) combining the `.container` utility (which centers via `margin-inline: auto`) with `.hero__content`'s own anchor margin caused the whole text block to render centered instead of right-anchored, and `text-align: end` was backwards (in RTL, `end` resolves to the left) — fixed by setting `margin-inline-start: 0` explicitly and switching to `text-align: start`.
- **Section 2 — Hero background photo** — Sourced the client-provided `hero.png` (dark BMW in a concrete showroom, raking key light — a strong match for the logged prompt) in as the real hero background, replacing the CSS-only gradient wash. Optimized with `sharp` to `assets/images/hero-showroom.{webp,jpg}` (1.85MB → 50KB/65KB); added a `<picture>` element with `fetchpriority="high"` plus a matching `<link rel="preload" as="image">` since this is the page's LCP element. Split the old single gradient into `.hero__scrim`, layered above the photo, tuned lighter than the original CSS-only wash since the source photo is already dark, plus an added end-side gradient to guarantee text contrast over the image's lighter zones. Deviation: source photo is 1408x768 (~1.83:1) rather than the requested 21:9 — compensated with `object-position: 25% center` so the car stays in frame at narrow/tall crops; see `IMAGE_PROMPTS.md` for the watermark-visibility check.
